# Campuses

Welcome to our Campuses web application! This project was built to demonstrate dynamic client-side routing, asynchronous server-state synchronization, responsive utility-first layouts, and seamless cloud hosting workflows.

The application is a clean, single-page dashboard featuring:
- **React Router Navigation:** Manages client-side routing with nested layout pages, query-driven student and campus detail views, and structured form paths alongside standard 404 page routing.
- **TanStack Query Integration:** Coordinates asynchronous server state using TanStack Query (React Query) to handle client-side query caching, loading/error state handling, automated retries, and instant cache invalidations on mutations.
- **Tailwind CSS Layouts:** Implements a utility-first styling system with responsive grids and flex layouts, a theme-adaptive dark mode, custom color gradients, and micro-interactive hover transitions.
- **Vercel Cloud Deployment:** Hosted live on Vercel utilizing configured rewrite rules to handle React Router client-side routing fallback logic seamlessly upon page refreshes or deep-linked entries.
- **Relational CRUD Dashboard:** Oversees linked databases for campuses and students, supporting complex enrollment relationships, registry profile grids, and dynamic visual fallbacks.

It is built using React, Vite, TypeScript, and Tailwind CSS (powered by TanStack Query for state sync and React Router for client-side view management, deployed on Vercel).
